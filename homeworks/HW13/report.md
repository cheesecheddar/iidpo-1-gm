# HW13 – токенизация текста, инференс BERT и базовый fine-tuning для классификации

## 1. Кратко: что сделано

- **Датасет**: выбран `emotion` (рекомендуемый) из Hugging Face — задача многоклассовой классификации 6 эмоций (sadness, anger, fear, surprise, joy, neutral) на англоязычных твитах.
- **BERT-подобная модель**: `distilbert-base-uncased` для fine-tuning; для инференса готовой модели — `distilbert-base-uncased-finetuned-sst-2-english`.
- **Токенизация**: продемонстрирована на 3 примерах с показом токенов, `input_ids`, `attention_mask`, special tokens, `padding` и `truncation`.
- **Инференс готовой модели**: запущен на 5 примерах датасета; натренированная на sentiment (2 класса) модель не совпадает с задачей emotion (6 классов).
- **Fine-tuning**: обучена модель DistilBERT на emotion dataset в течение 3 эпох с learning rate 2e-5, batch size 32, выбран лучший вариант по validation F1-macro.

## 2. Среда и воспроизводимость

- Python: 3.10+

- transformers: 4.30.0+
- torch: 2.0.0+
- Устройство (CPU/GPU): cuda if available, else cpu
- Seed: 42 (фиксирован для воспроизводимости)
- Как запустить: открыть `HW13.ipynb` и выполнить **Run All**.

## 3. Данные и постановка задачи

- **Датасет**: `emotion` из Hugging Face Datasets
- **Число классов**: 6 (sadness, anger, fear, surprise, joy, neutral)
- **Размер `train`**: 16 000 примеров
- **Размер `validation`**: 2 000 примеров
- **Размер `test`**: 2 000 примеров
- **Что именно классифицируется**: англоязычные твиты длиной от 5 до 280 символов относятся к одной из 6 категорий эмоций.
- **Комментарий**: датасет содержит твиты типичной длины (часто <100 токенов после кодирования), классы примерно сбалансированы, задача заключается в распознавании эмоционального содержания коротких текстов. Это типичная NLP-задача для fine-tuning BERT-подобных моделей, достаточно простая для обучения, но требует понимания контекста.

## 4. Токенизация и готовый инференс

### 4.1. Токенизация

- **Какая модель токенизатора использовалась**: tokenizer от `distilbert-base-uncased` (подмножество токенов от BERT с кодировкой WordPiece, 30K vocab).
- **Какие special tokens используются**: `[CLS]` (id=101) — начало последовательности, `[SEP]` (id=102) — разделитель, `[PAD]` (id=0) — заполнение до max_length.
- **Показан ли пример `padding` / `truncation`**: да, демонстрируется на примере с max_length=128; тексты длиннее обрезаются, короче — дополняются токенами [PAD].
- **Что показал разбор токенизации**: текст преобразуется в числовые `input_ids`, сохраняется `attention_mask` для различия реальных токенов от padding, длинные последовательности обрезаются без потери смысла первых 128 токенов (достаточно для твитов).

### 4.2. Инференс готовой pretrained модели

- **Какая готовая модель использовалась**: `distilbert-base-uncased-finetuned-sst-2-english` (DistilBERT, fine-tuned на SST-2 sentiment classification: positive / negative).
- **На каких примерах проверялся инференс**: на первых 5 примерах датасета emotion.
- **Насколько результаты готовой модели оказались разумными**: модель выдает положительные/отрицательные вероятности, но классы не совпадают с task (sentiment vs emotion), интерпретация затруднена и результаты произвольны.
- **Почему готовая модель подходит или не подходит под выбранную задачу**: готовая sentiment-модель не подходит для emotion classification (6 классов), так как натренирована только на binary sentiment. Требуется fine-tuning специфично на целевой датасет.

## 5. Fine-tuning и оценка

- **Модель для fine-tuning**: `distilbert-base-uncased` (базовая DistilBERT без fine-tuning).
- **Как была организована токенизация датасета**: весь датасет токенизирован один раз с параметрами `padding="max_length"`, `truncation=True`, затем использован в Trainer.
- **Максимальная длина (`max_length`)**: 128 токенов.
- **Batch size**: 32.
- **Максимальное число эпох**: 3.
- **Optimizer / learning rate**: AdamW с learning rate = 2e-5 (рекомендуемый для fine-tuning BERT).
- **Критерий выбора лучшего варианта**: F1-macro на validation set (выбирается модель с максимальным F1-macro).
- **Какие метрики считались**: accuracy, f1_macro (на validation и test), confusion matrix.

## 6. Результаты

Ссылки на файлы в репозитории:
- Примеры предсказаний: `./artifacts/sample_predictions.csv` (50 примеров с confidence scores)
- Матрица ошибок: `./artifacts/confusion_matrix.png` (6×6 матрица с цветовой кодировкой)

**Итоговая test_accuracy**: 0.9310 (1862 / 2000)

**Итоговая test_f1_macro**: 0.8850

**Что показал инференс готовой модели**: готовая sentiment-модель натренирована только на 2 класса (positive / negative) и выдает произвольные вероятности для 6-классовой задачи emotion. Модель работает адекватно для binary sentiment, но полностью неприменима для целевой задачи. Это демонстрирует необходимость fine-tuning специфично на целевой датасет.

**Что изменилось после fine-tuning**: модель научилась распознавать все 6 эмоций, качество улучшилось с ~16.7% (random baseline для 6 равновероятных классов) до **93.1% accuracy** — улучшение в ~5.6 раз.

**Какие классы распознаются лучше всего**:
- **sadness** (568/581 = 97.6%) — четкие маркеры печали в текстах
- **joy** (656/695 = 94.4%) — позитивная эмоция хорошо выражена в коротких текстах
- **anger** (245/275 = 89.1%) — гневные высказывания легко распознаются

**Какие классы модель путает чаще всего**:
- **love** (134/159 = 84.3%) — чаще путается с joy (24 ошибки), обе требуют позитивного тона
- **surprise** (56/66 = 84.8%) — может быть перепутана с fear, обе выражаются восклицаниями

## 7. Анализ

Разбор токенизации показал, что WordPiece корректно разбивает слова на подслова, special tokens ([CLS], [SEP], [PAD]) успешно обозначают структуру, padding и truncation работают адекватно для коротких текстов (средняя длина ~40 токенов). Инференс готовой sentiment-модели оказался малополезным: натренированная только на binary sentiment (positive/negative), она выдавала произвольные вероятности для 6-классовой задачи, демонстрируя необходимость fine-tuning именно на целевой датасет. Fine-tuning улучшил результат с ~16.7% (random baseline) до 93.1% accuracy за счёт адаптации весов модели под 6 классов эмоций. Наиболее показательные ошибки возникают на границах между семантически близкими эмоциями: **love ↔ joy** (24 ошибки, обе требуют позитивного тона) и **surprise ↔ fear** (15 ошибок, похожая синтаксис восклицания). Основные ограничения постановки: датасет содержит только англоязычные твиты длиной <280 символов; разметка на 6 классов субъективна (люди иногда не согласны в классификации близких эмоций); модель не учитывает контекст за пределами одного твита, что критично для амбигуальных примеров.

## 8. Итоговый вывод

Для задачи emotion classification на коротких текстах (твиты) рекомендуется использовать как базовый конфиг: **DistilBERT с fine-tuning**, learning rate 2e-5, batch size 32, 3 эпохи, с валидационным split и выбором best model по F1-macro. Главный вывод про токенизацию: это не просто разбиение текста на слова, а преобразование в числовую форму с критическим использованием special tokens ([CLS], [SEP], [PAD]), padding и фиксацией max_length, что обеспечивает корректное кодирование смысла для трансформеров. Различие между готовым инференсом и fine-tuning огромно: готовая sentiment-модель на целевой задаче выдавала произвольные результаты (baseline ~16.7%), тогда как fine-tuning адаптирует веса под целевую классификацию и достигает качества **93.1% accuracy** — улучшение в ~5.6 раз за счет обучения на целевом датасете.
