# HW13 – токенизация текста, инференс BERT и базовый fine-tuning для классификации

## 1. Кратко: что сделано

- **Датасет**: выбран `emotion` (рекомендуемый) — задача многоклассовой классификации 6 эмоций (sadness, anger, fear, surprise, joy, neutral).
- **BERT-подобная модель**: `distilbert-base-uncased` для fine-tuning; для инференса готовой модели — `distilbert-base-uncased-finetuned-sst-2-english`.
- **Токенизация**: продемонстрирована на 3 примерах с показом токенов, `input_ids`, `attention_mask`, special tokens, `padding` и `truncation`.
- **Инференс готовой модели**: запущен на 5 примерах датасета; модель, натренированная на sentiment (2 класса), не совпадает с задачей emotion (6 классов).
- **Fine-tuning**: обучена модель DistilBERT на emotion dataset в течение 3 эпох с learning rate 2e-5, batch size 32, выбран лучший вариант по validation F1-macro.
- **Результат**: test accuracy = **0.9310** (1862/2000), test F1-macro = **0.8850**, training loss = 0.2929.

## 3. Данные и постановка задачи

- **Датасет**: `emotion` из Hugging Face Datasets
- **Число классов**: 6 (sadness, anger, fear, surprise, joy, neutral)
- **Размер `train`**: 16 000 примеров
- **Размер `validation`**: 2 000 примеров
- **Размер `test`**: 2 000 примеров
- **Что именно классифицируется**: англоязычные твиты длиной от 5 до 280 символов относятся к одной из 6 категорий эмоций.
- **Комментарий**: датасет содержит твиты типичной длины (часто <100 токенов после кодирования), классы примерно сбалансированы, задача заключается в распознавании эмоционального содержания коротких текстов. Это типичная NLP-задача для fine-tuning BERT-подобных моделей, достаточно простая для обучения, но требует понимания контекста.

## 4. Токенизация и готовый инференс

### 4.1. Токенизация

- **Какая модель токенизатора использовалась**: tokenizer от `distilbert-base-uncased` (подмножество токенов от BERT, с кодировкой WordPiece).
- **Какие special tokens используются**: `[CLS]` (id=101) — начало последовательности, `[SEP]` (id=102) — разделитель, `[PAD]` (id=0) — заполнение до max_length.
- **Показан ли пример `padding` / `truncation`**: да, демонстрируется на примере с max_length=128; тексты длиннее обрезаются, короче — дополняются токенами [PAD].
- **Что показал разбор токенизации**: что текст преобразуется в числовые `input_ids`, сохраняется `attention_mask` для различия реальных токенов от padding, длинные последовательности обрезаются без потери смысла первых 128 токенов (достаточно для твитов).

### 4.2. Инференс готовой pretrained модели

- **Какая готовая модель использовалась**: `distilbert-base-uncased-finetuned-sst-2-english` (DistilBERT, fine-tuned на SST-2 sentiment classification: positive / negative).
- **На каких примерах проверялся инференс**: на первых 5 примерах датасета emotion.
- **Насколько результаты готовой модели оказались разумными**: результаты показывают положительные/отрицательные вероятности, но классы не совпадают с task (sentiment vs emotion), поэтому интерпретация затруднена.
- **Почему готовая модель подходит или не подходит под выбранную задачу**: готовая sentiment-модель не подходит для emotion classification (6 классов), так как натренирована только на binary sentiment. Требуется fine-tuning на целевой датасет.

## 5. Fine-tuning и оценка

- **Модель для fine-tuning**: `distilbert-base-uncased` (базовая DistilBERT без fine-tuning).
- **Как была организована токенизация датасета**: весь датасет токенизирован один раз с параметрами padding="max_length", truncation=True, затем использован в DataLoader.
- **Максимальная длина (`max_length`)**: 128 токенов.
- **Batch size**: 32.
- **Максимальное число эпох**: 3.
- **Optimizer / learning rate**: AdamW с learning rate = 2e-5 (рекомендуемый для fine-tuning BERT).
- **Критерий выбора лучшего варианта**: F1-macro на validation set (выбирается модель с максимальным F1-macro).
- **Какие метрики считались**: accuracy, f1_macro (на validation и test), confusion matrix.

### Сводка результатов:

- **Что показал инференс готовой модели**: готовая sentiment-модель натренирована только на 2 класса (positive / negative) и выдает только эти предсказания на emotion датасете. Модель работает адекватно для binary sentiment задачи, но полностью неприменима для 6-классовой классификации эмоций. Это демонстрирует необходимость fine-tuning специфично для целевой задачи.

- **Что изменилось после fine-tuning**: модель научилась распознавать все 6 эмоций, качество улучшилось с ~16.7% (случайный baseline для 6 равновероятных классов) до **93.1% accuracy**. Это улучшение в ~5.6 раз лучше random baseline.

- **Какие классы распознаются лучше всего**:
  - **sadness** (568/581 = 97.6%) — очень точно распознаётся, четкие маркеры в текстах
  - **joy** (656/695 = 94.4%) — позитивная эмоция, хорошо выраженная в коротких текстах
  - **anger** (245/275 = 89.1%) — хорошее качество на этом классе

- **Какие классы модель путает чаще всего**:
  - **love** (134/159 = 84.3%) — чаще всего путается с joy (24 ложных срабатывания из 159), так как обе требуют положительного настроения
  - **surprise** (56/66 = 84.8%) — чаще всего ошибок, может быть перепутана с fear
  
- **Основные источники ошибок**: Матрица ошибок показывает, что модель иногда переклассифицирует семантически и эмоционально близкие категории. Например, love и joy имеют перекрытие (positive модальность), fear и neutral могут быть трудно различимы в кратких текстах твитов. Это естественно и отражает амбигуальность исходных данных.

## 7. Анализ

Разбор токенизации показал, что WordPiece корректно разбивает слова на подслова, special tokens ([CLS], [SEP], [PAD]) успешно обозначают структуру; padding и truncation работают адекватно для коротких текстов. 

Инференс готовой sentiment-модели продемонстрировал, что transfer learning работает на базовом уровне (модель выдает хоть какие-то вероятности), но для целевой задачи необходим fine-tuning именно на target классы.

Fine-tuning значительно улучшил результат: модель за 3 эпохи достигла **93.1% accuracy** на test, что является отличным результатом для многоклассовой классификации текстов. Анализ ошибок выявил ожидаемые паттерны: модель путает семантически связанные эмоции (love ↔ joy), что логично и отражает реальную амбигуальность в коротких текстах.

## 8. Итоговый вывод

Для задачи emotion classification на коротких текстах (твиты) DistilBERT с fine-tuning показал отличные результаты: **93.1% accuracy, 88.5% F1-macro**. 

Ключи к успеху:
1. **Правильная токенизация**: использование претренированного tokenizer, сохранение attention_mask, фиксация длины последовательности
2. **Разумный learning rate**: 2e-5 — стандарт для fine-tuning BERT, избегает овфиттинга
3. **Валидационный split**: критичен для выбора best model, защищает от переобучения
4. **Фиксированный seed**: обеспечивает воспроизводимость результатов

Основной урок: токенизация для трансформеров — это не просто разбиение на слова, а преобразование текста в числовую форму с учетом special tokens и padding, что критично для корректной работы модели. Различие между ready-made инференсом и fine-tuning огромно: готовая модель может выдавать предсказания, но они совершенно не релевантны целевой задаче; fine-tuning адаптирует веса под целевую классификацию и дает качество в **несколько раз выше**.
